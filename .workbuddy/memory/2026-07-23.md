## 第二轮重设计 —— 大图标卡片 + 去阴影播放按钮

### 用户反馈
- 上一版图标 h-6 (24px) 装在 h-12 (48px) 圆里太空太小
- 卡片纯白太素，缺少视觉冲击
- 播放按钮绿色 box-shadow 突兀，要求去掉阴影
- 播放按钮 icon "不好搞"，要求复用简洁原始形式

### 改动 (`src/pages/index/index.vue`)
- **卡片背景**: 未选中 = 主题色 14% 透明 (`#color14`)，选中 = 主题色 100% 实色
- **大图标**: h-14 w-14 (56px)，白色或主题色
- **固定高度**: 210rpx 让卡片视觉对齐
- **选中标识**: 4rpx 主题色边框 + 右上角白色半透明对勾圆标
- **播放按钮**: 删除 `box-shadow: 0 4px 16px rgba(7,193,96,0.35)`，纯绿圆 + 白色 play/pause
- **底部三键阴影**: 从 `shadow-lg` 改为 `0 2rpx 12rpx rgba(0,0,0,0.04)` 极淡
- **背景渐变**: 底部栏上方用 `linear-gradient(to top, #FFF8F2 70%, transparent)` 让按钮浮起
- **布局**: 音量面板和底部三键改 `absolute` 浮动，scroll-view 用 `pb-32` 留空间

### 播放按钮 icon
- `i-carbon-play-filled` → `i-carbon-play-filled-alt`（更标准三角形）

### Carbon 图标缺失与替代（已修复）

## 多轨混音 + 图片映射 + 单独音量（第三轮）

### 用户需求
- 把 `src/static/image/` 下的图片与音频做关联，没有图片的留空
- 小程序改成类似 asoftmurmur.com：可同时播放多个音频，每个音频独立音量
- 重新设计首页为混音器界面

### 图片映射结果（8 张图对应 8 个音频）
| 图片 | 音频 |
|------|------|
| bird.png | 鸟鸣 |
| cloud-lightning.png | 雷声 |
| cloud-rain-wind.png | 雨声 |
| flame.png | 篝火声 |
| leaf.png | 蟋蟀声 |
| tv.png | 白噪音 |
| waves-horizontal.png | 海浪声 |
| wind.png | 风声 |

**无图**：颂钵、咖啡厅 → `image: ""`

### 音频架构变更
- 从单轨 `BackgroundAudioManager` 改为多轨 `InnerAudioContext`
- 新增 `src/utils/audioMixer.ts`：管理 `Map<id, InnerAudioContext>`，每轨独立 `volume` 和 `loop`
- 删除已废弃的 `src/utils/audio.ts`
- **权衡说明**：微信小程序里多轨混音只能用 `InnerAudioContext`，后台播放会被系统暂停；如需后台播放必须切回单轨 `BackgroundAudioManager`。

### Store 改造 (`src/store/player.ts`)
- `activeSounds: Record<id, volume>` 替代 `currentSoundId`
- 新增 `activeSoundList` / `activeSoundItems` computed
- `toggleSound(id)`：添加/移除音轨
- `setSoundVolume(id, volume)`：单独调节某一轨音量
- `playAllActive()`：播放所有已激活音轨
- `initFromPersist()`：从 Pinia 持久化恢复音轨实例

### UI 改造 (`src/pages/index/index.vue`)
- 卡片高度从 210rpx 提升到 280rpx
- 每个卡片顶部显示 `image`（无图则回退 carbon icon）
- **激活态**：主题色实底 + 白色文字 + 白色音量滑块
- **未激活态**：白色卡片 + 主题色半透明边框 + "点击添加" 小标签
- 底部播放按钮在无选中音源时变灰，禁用感

### 数据改动 (`src/static/sounds.json` + `src/types/sound.ts`)
- `SoundItem` 新增 `image: string` 字段
- 每个声音配置 `image` 路径
- 版本号 `version: 4`

### 图标修复
- `i-carbon-waves` 不存在 → `i-carbon-wave-direction`
